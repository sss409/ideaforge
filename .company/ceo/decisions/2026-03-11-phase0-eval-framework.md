# Phase 0 評価フレームワーク策定

## 意思決定

Phase 0 クリア判定のための評価フレームワークを以下の通り策定する。

## 内容

1. **評価ルブリック v1** を作成（5軸 × 5段階 = 25点満点）
2. **LLM-as-judge プロンプト v1** を作成（Sonnet 4.6 による自動採点）
3. **テストアイデアを 20件に拡充**（v1: 5件 → v2: +15件）
4. **Phase 0 クリア基準**: 平均 20/25 以上 かつ チェック項目に ❌ なし

## 理由

- 定量評価がなければ「品質OK」の判断が主観に依存する
- LLM-as-judge で自動化することで、プロンプト改善の反復を高速化
- 20件のテストで多様なドメイン・情報量をカバー

## 次のアクション

- 開発: Idea 2, 4, 5 で Step 1 v1.1 → Step 2 v1.1 をテスト
- 開発: LLM-as-judge を使ってスコアリング
- PM: テスト結果をスコアカードに記録
