# Idea 9 テスト結果: 空き家DIYリノベマッチング

## 元の入力

> 空き家所有者と、DIYリノベしたい若者をマッチングするプラットフォーム。所有者は固定資産税の負担軽減、若者は格安で住居を確保。自治体の空き家バンクは使いにくいので、UXを改善してマッチング精度を上げる。成約時に仲介手数料5%。

---

## Step 1: 構造化（Structurize）

### モード判定の分析

入力から推論可能なフィールド:
- **domain**: 推論可能（不動産）
- **problem**: 推論可能（空き家の増加と所有者の税負担、若者の住居コスト問題。自治体の空き家バンクのUXが悪くマッチングが機能していない）
- **solution**: 推論可能（空き家所有者とDIYリノベ希望の若者をマッチングするプラットフォーム。UX改善とマッチング精度向上）
- **target_users**: 推論可能（空き家所有者、DIYリノベしたい若者）
- **differentiators**: 部分的に推論可能（自治体空き家バンクとの差別化は明示されているが、民間競合との差別化は不明確）

→ 推論困難なフィールド: differentiators のみ（部分的） = 1つ → **モード A**

### Step 1 チェックリスト

| チェック項目 | 結果 | 備考 |
|-------------|------|------|
| モード判定（A/B）が正しいか | PASS | 5フィールド中4つ以上が推論可能。Mode A は正しい |
| title をユーザーに質問していないか | PASS | AIが「空き家DIYリノベマッチング」を生成 |
| target_users を入力外に勝手に拡大していないか | PASS | 入力に明示された「空き家所有者」と「DIYリノベしたい若者」の2グループのみ。自治体担当者や不動産業者等は追加していない |
| differentiators が3個あるか | PASS | 3個あり。UX・体験設計、ビジネスモデル、ターゲット特化の3軸から1つずつ |
| differentiators が2つ以上の軸カテゴリから出ているか | PASS | 3つの異なる軸（UX・体験設計 / ビジネスモデル / ターゲット特化）を使用 |
