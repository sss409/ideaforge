# Idea 10 テスト結果: 中小企業の採用がうまくいかない

## 元の入力

> 中小企業の採用がうまくいかない。何かいいツールないかな

---

## Step 1: 構造化（Structurize）

### モード判定の分析

入力から推論可能なフィールド:
- **domain**: 推論可能（HRテック）
- **problem**: 部分的に推論可能（「採用がうまくいかない」とあるが、母集団形成なのか、選考効率なのか、内定辞退なのか、採用ブランディングなのかが不明）
- **solution**: 推論困難（「何かいいツール」のみで、ATS（採用管理システム）なのか、求人媒体なのか、スカウトツールなのか、AI面接なのかが全く不明）
- **target_users**: 部分的に推論可能（「中小企業」は明示されているが、業種・規模・採用職種・採用人数が不明）
- **differentiators**: 推論困難（既存ツールとの差別化要素が全く示されていない）

→ 推論困難なフィールド: solution + differentiators + problem（具体性不足）+ target_users（具体性不足） = 4つ → **モード B**

### Step 1 チェックリスト

| チェック項目 | 結果 | 備考 |
|-------------|------|------|
| モード判定（A/B）が正しいか | PASS | solution + differentiators が推論困難、problem + target_users も具体性不足 = 4フィールド推論困難 → モード B は正しい |
| title をユーザーに質問していないか | PASS | title への質問なし。3つの質問は problem, solution, differentiators を対象としている |
| target_users を入力外に勝手に拡大していないか | PASS | understood の target_users は入力通り「中小企業（採用担当者または経営者）」。業種や規模を勝手に特定していない |
| differentiators が3個あるか | PASS | モード B のため differentiators は出力不要。質問で確認する設計は適切 |
| 質問がドメイン固有の課題に言及しているか | PASS | HRMOS、ジョブカン、Talentio等のATS名、ハローワーク・Indeed等の求人媒体名、母集団形成・内定辞退等のHR固有用語を使用 |
