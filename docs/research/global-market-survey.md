# 海外「アイデアを売る」市場 調査レポート

調査日: 2026-03-11

## 結論

1. **「アイデアだけ」は売れない。** 英語圏の圧倒的コンセンサス
2. **売れるのは「アイデア + 何か」。** 最低でも動くプロトタイプ or 検証データが必要
3. **AI/Vibe Coding で「アイデアの人」が自分で作れる時代になった。** アイデア売買の必要性自体が変質
4. **Sam Altman: 「Idea Guyの時代が来る」** — ただし「自分で作れるから」であって「売れるから」ではない
5. **検証ツール単体は小規模市場。** DimeADozen: $8K/月で$150Kで売却。MVP生成ツール（Lovable $300M ARR）の100分の1以下

## 2. 成功パターン

### パターンA: AI検証ツール → 売却（DimeADozen）
- 開発コスト$185 → 7ヶ月で$66K収益 → **$150Kで売却**
- 成功要因: AIニュースレターからの流入（60%リファラル）、Product Huntローンチ
- 限界: $8K/月で頭打ち。市場規模が小さい

### パターンB: MVP生成ツール（Lovable/Bolt）
- 「アイデア→プロダクト」の変換を自動化
- 市場規模は検証ツールの100倍以上
- **検証と実装の間のギャップ**がビジネスチャンス

### パターンC: 問題発掘プラットフォーム（Buildpad/StartupIdeasDB/ProblemHunt）
- Reddit等をスキャンして「本当のペインポイント」を発見
- 「アイデアを売る」のではなく「検証済み問題を売る」
- Buildpad: $0→$7.3K/月を7ヶ月で達成

### パターンD: プレレベニュープロジェクト売却
- Acquire.comでプレレベニュー平均$12,695で売却
- Flippaでゼロレベニューサイトが$2K-5Kで取引
- 条件: 動くコード + ドメイン + (あれば)メールリスト

## 3. 失敗パターン

### 「アイデアだけ」を売るマーケットプレイス → 全滅
- IdeaMarket: 閉鎖（「勢いを得られなかった」）
- Idea Marketplace: 閉鎖
- 失敗率: マーケットプレイスの42%が「市場ニーズなし」で失敗

### AI検証ツールの共通問題
1. **データ捏造**: 市場規模等の数値がプレースホルダー（VentureKit）
2. **確認バイアス**: 全てのアイデアに高スコアを出す（「大量破壊兵器会社」に65/100）
3. **ChatGPTラッパー**: OpenAIがAPI止めたら死ぬビジネス
4. **差別化不能**: 同じAPIを使う以上、出力品質に差がない

### 発明ライセンシング → 詐欺だらけ
- Edison Nation: 成功率0.04%
- FTC: 年間$300Mの詐欺被害
- 「あなたのアイデアには市場ポテンシャルがある」と全員に言う → 高額請求

## 4. AI/Vibe Coding による構造変化（2025-2026）

### 「アイデアの人」の定義が変わった

| 従来（〜2024） | 現在（2025〜） |
|---------------|---------------|
| アイデアだけ持っている | AIで自分でプロトタイプを作れる |
| 開発者を探す必要がある | 28-45分でMVP完成 |
| アイデアを「売る」しかない | アイデアを「作って売る」ことが可能 |

### 数字で見る変化
- Replit: 75%のユーザーがコードを一行も書いていない
- YC 2025冬: 25%のバッチがコードの95%以上をAI生成
- 米国: 新規スタートアップの35%がソロ創業者（2025年）
- Collins Dictionary: 「vibe coding」が2026年の Word of the Year
- 市場規模: 2025年$4.7B → 2027年$12.3B予測

### Sam Altman「Idea Guyの時代」
> 「我々は良いアイデアによって制約される（We will be limited by good ideas）」
- AIが安価になるにつれ、希少資源は実装力→発想力に移行
- ただしAI自体がアイデアを生成できるようになれば、この優位性は一時的

---

## 5. ユーザーの未充足ニーズ（競合の弱み）

既存ツールに対するユーザーの不満:
1. **実データに基づいていない** — AI生成の推測値しかない
2. **「やめろ」と言ってくれない** — 常にポジティブ（確認バイアス）
3. **創業者のスキル/資金を考慮しない** — 万人向けの汎用レポート
4. **検証で終わる** — 次のステップ（プロトタイプ、LP、需要テスト）につながらない
5. **検証→実装のギャップ** — 検証ツールと実装ツールが分断されている
