# IdeaForge 開発アプローチ調査結果

調査日: 2026-03-11

## 調査概要

IdeaForge に最適な開発アプローチを3軸で調査:
1. AIマーケットプレイスのMVP開発手法
2. B2Bマーケットプレイスの検証手法
3. Claude API の本番活用パターン

## 核心的発見: 「コードを書く前に売れ」

### 成功事例のMVP戦略

| 事例 | やったこと | 結果 |
|------|----------|------|
| Jasper AI | 30日でMVP、GPT APIラッパーに集中 | 3週目で月商$20K、9ヶ月でARR $40M |
| SmartHR | 半日でLP（WordPress 3,000円）、FB広告2万円 | 3日で100件プレ登録 |
| Airbnb | WordPressブログ、予約はコメント欄 | コードなしで需要確認 |
| DoorDash | チラシ+電話、アプリなし | 手動配達で検証 |
| Stripe | 「ノートPC貸して」→その場でセットアップ | Collison Installation |

### YC の教訓

- 「スケールしないことをしろ」（Paul Graham）
- 最初の10顧客は個人ネットワークから（Michael Seibel）
- 初日から課金しろ — 支払いが最強の需要検証
- 8週間以内にローンチ — ローンチ前の成長率はゼロ

## 推奨フェーズ計画

### Phase 0: プロンプト開発（1-2週）

- コードを書かない
- Claude Web UI で精錬プロンプトを完成
- 20-30件のテストアイデアで品質検証
- 評価ルブリック + LLM-as-judge

### Phase 1: LP + 手動コンシェルジュ（2-4週）

- ランディングページ + Typeform + Stripe Payment Links
- 手動でアイデアを精錬（Wizard of Oz 方式）
- 検証ゲート: 3-5件の実購入がなければ Phase 2 に進まない

### Phase 2: MVP 開発（4-6週）

- 検証済み機能のみ
- Next.js + Supabase + Clerk + Stripe Connect + Claude API

## 技術選定の根拠

### Supabase（PostgreSQL + RLS）

- Row Level Security が3層開示モデルに最適
- 無料枠で MVP 十分
- pgvector 対応済み

### Clerk（認証）

- 15分でセットアップ完了
- SOC2 Type II 対応（B2B必須）
- 10K MAU まで無料
- NextAuth.js より大幅に少ない実装量

### Stripe Connect（決済）

- 3者間マーケットプレイス対応（発案者→PF→買い手）
- Phase 1 では Payment Links（コード不要）で十分

### Claude API 直接（LangChain不要）

- 5ステップ固定パイプラインに抽象化レイヤーは過剰
- 構造化出力（Pydantic）で型安全
- プロンプトキャッシュで90%コスト削減
- ~100行の Python で十分なオーケストレーション

## Claude API 本番パターン

### コスト見積もり

| ボリューム | リアルタイム | キャッシュ適用後 |
|-----------|-----------|----------------|
| 100件/月 | ~$18 | ~$3-5 |
| 1,000件/月 | ~$180 | ~$30-50 |
| 10,000件/月 | ~$1,800 | ~$300-500 |

### 品質保証

1. コードベースの検証（スキーマ準拠、必須フィールド）
2. LLM-as-judge（Sonnet で5軸評価）
3. 人間レビュー（低スコアのルーティング + 5-10%サンプリング）

## 日本市場の B2B 注意点

- 稟議制度: 月額固定 + クレジット制のハイブリッド価格
- 営業サイクル 8-12ヶ月: 初期はSMB、30日有料パイロット
- SmartHR方式の価格検証: 間接質問「いくらだと高すぎますか？」
